O que e o ParcerIA?

Plataforma SaaS para gestao de acesso, escalas medicas e produtividade de trabalhadores terceirizados nas unidades hospitalares parceiras da Agir Saude.

Parceria + IA = Integracao com Inteligencia Artificial para analise de dados e insights automatizados.

🏥

🤖

📊

Tempo Real

Dados de catracas biometricas atualizados constantemente

ParcerIA 04 / 20

Problema que Resolvemos

Antes do ParcerIA

  • Dados fragmentados em sistemas separados
  • Sem visao consolidada de horas trabalhadas
  • Importacao manual via planilhas Excel
  • Analise demorada e propensa a erros
  • Sem insights ou analise preditiva

Com o ParcerIA

  • Dados unificados em uma plataforma
  • Dashboard em tempo real
  • Importacao em lote via CSV
  • Calculos automaticos de horas
  • IA gerando insights diarios

ParcerIA 10 / 20

Supabase vs. AWS: Comparativo Tecnico

Recurso Supabase Equivalente AWS Explicacao
Banco de Dados PostgreSQL RDS PostgreSQL Ambos oferecem banco relacional robusto. RDS e o servico gerenciado da Amazon.
Autenticacao Auth (JWT) Cognito User Pools Supabase usa JWT nativo. Cognito gerencia login e perfis de usuarios.
Armazenamento Storage S3 Ambos guardam arquivos (PDFs, fotos). S3 e o padrao da industria.
Logica Serverless Edge Functions Lambda + API Gateway Edge Functions = codigo sem servidor. Na AWS, combina Lambda + API Gateway.
Tempo Real Realtime AppSync / WebSocket Atualiza dados instantaneamente sem reload (como um chat).
Seguranca RLS IAM + Middleware Supabase protege no banco. AWS usa IAM + codigo intermediario.
IA / Vetores pgvector RDS + extensao Ambos armazenam vetores para busca semantica. Na AWS, habilitar manualmente.

ParcerIA 13 / 20

Fase 1-2: Migracao do Banco

Passos

  • Criar RDS PostgreSQL 13+ Multi-AZ
  • Habilitar extensao pgvector
  • Exportar dados com pg_dump
  • Importar no RDS com pg_restore
  • Executar 25 migrations
# Exportar do Supabase pg_dump -h db.xxx.supabase.co \ -U postgres -F c -f backup.dump # Importar no RDS pg_restore -h parceria.rds.aws.com \ -U admin -d parceria backup.dump # Habilitar pgvector CREATE EXTENSION vector;

PostgreSQL e PostgreSQL! Schema, tabelas, queries migram sem alteracao.

ParcerIA 14 / 20

Fase 3: Migracao da Autenticacao

ANTES (Supabase)

import { supabase } from './supabase' const { data } = await supabase.auth.signInWithPassword({ email, password })
->

DEPOIS (Cognito)

import { signIn } from 'aws-amplify/auth' const { isSignedIn } = await signIn({ username: email, password })

Migracao lazy: Lambda trigger autentica no Supabase na 1a vez, cria no Cognito.

ParcerIA 15 / 20

Fase 5-6: Edge Functions -> Lambda

ANTES (Deno)

serve(async (req) => { const { data } = await supabase .from('acessos') .select('*') return new Response( JSON.stringify(data) ) })
->

DEPOIS (Node.js)

exports.handler = async () => { const result = await client .query('SELECT * FROM acessos') return { statusCode: 200, body: JSON.stringify(result.rows) } }

Mesma logica, sintaxe similar. Deno ~ Node.js

Reimplementando RLS como Middleware

O maior desafio: RLS do Supabase vira middleware nas Lambdas.

// middleware/authorize.js exports.authorize = async (event, allowedTypes) => { // 1. Extrair e validar token JWT do Cognito const token = event.headers.Authorization const decoded = await verifyToken(token) // 2. Buscar perfil do usuario no banco const user = await client.query( 'SELECT tipo, unidade_id FROM usuarios WHERE id = $1', [decoded.sub] ) // 3. Verificar permissao e retornar contexto if (!allowedTypes.includes(user.tipo)) return { authorized: false } return { authorized: true, user } // user.unidade_id filtra queries }

ParcerIA 18 / 20

Perguntas Frequentes

Precisa recriar a logica de negocio?
NAO! Schema, queries, validacoes, componentes - tudo permanece. So a camada de conexao muda.
E as senhas dos usuarios?
Migracao lazy: Lambda autentica no Supabase na 1a vez, depois cria no Cognito.
O frontend muda muito?
Apenas arquivos de servico (supabase.ts -> aws.ts). Componentes, hooks e paginas ficam iguais.
E se der errado?
Rollback via feature flag. Manter Supabase ativo durante migracao.
